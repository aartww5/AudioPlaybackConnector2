# Release v0.5.3

## Summary

This release targets the two-device disconnect/reconnect crash reported in [issue #6](https://github.com/N0ahTM/AudioPlaybackConnector2/issues/6). It also includes the App Installer update-button fix from `0.5.2` follow-up work.

## Highlights

- **#6 crash fix:** Connection metadata is cached as plain strings; cascade restore uses `ConnectAsync`; settings saves are deferred off the UI path.
- **Update UX:** Settings can open the `.appinstaller` file directly when `ms-appinstaller:` is disabled.
- **Diagnostics:** Fatal crashes still write a short in-memory diagnostic tail to the log file (no extra noise during normal use).

## Test plan

1. Install `v0.5.3` over an existing MSIX install (or clean install with `.cer` + `.appinstaller`).
2. Connect **two** Bluetooth audio devices.
3. Disconnect one device from the tray picker.
4. Reconnect that device.
5. Repeat steps 3–4 for the other device.
6. Confirm the app stays running and both devices can be used again.

If anything still fails, attach `AudioPlaybackConnector2.log` (look for `BEGIN in-memory diagnostic tail` at the end) and note the app version **0.5.3**.

## Publishing checklist (maintainer)

1. Merge `develop` (or release branch) with these changes committed.
2. Create and push tag: `git tag v0.5.3` && `git push origin v0.5.3`
3. Wait for the [Release workflow](https://github.com/N0ahTM/AudioPlaybackConnector2/actions/workflows/release.yml) to build the signed MSIX, publish the App Installer feed, and finalize the GitHub Release.
4. Comment on issue #6 with the release link and the test plan above.

## Runtime dependencies

Install once before the MSIX:

1. [WinAppRuntime.Singleton](https://apps.microsoft.com/detail/9p5z076k079h)
2. [Windows App SDK 2.0 runtime](https://learn.microsoft.com/en-us/windows/apps/windows-app-sdk/downloads#windows-app-sdk-20)

## Installation

```powershell
Import-Certificate -FilePath ".\AudioPlaybackConnector2.cer" -CertStoreLocation "Cert:\LocalMachine\Root"
Start-Process "ms-appinstaller:?source=https://n0ahtm.github.io/AudioPlaybackConnector2/AudioPlaybackConnector2.appinstaller"
```

If the protocol link does not open, download `AudioPlaybackConnector2.appinstaller` from the release assets and open it locally.
